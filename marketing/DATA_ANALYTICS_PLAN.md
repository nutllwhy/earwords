### 四、用户分层指标

#### 4.1 RFM分层模型

```
R - Recency（最近学习）
├── R5：1天内学习
├── R4：2-3天内学习
├── R3：4-7天内学习
├── R2：8-30天内学习
└── R1：30天以上未学习

F - Frequency（学习频率）
├── F5：周学习7天
├── F4：周学习5-6天
├── F3：周学习3-4天
├── F2：周学习1-2天
└── F1：周学习0天

M - Monetary（付费金额）
├── M5：年费会员
├── M4：季费会员
├── M3：月费会员
├── M2：单次付费
└── M1：免费用户

用户分层：
- 核心用户：R5 + F5 + M5/M4
- 重要用户：R4 + F4 + M3/M2
- 潜力用户：R5 + F3 + M1
- 风险用户：R2/R1 + F2/F1 + 任意M
- 流失用户：R1 + F1
```

#### 4.2 生命周期分层

| 用户层级 | 定义 | 特征 | 运营策略 |
|----------|------|------|----------|
| 新用户 | 注册7天内 | 刚接触产品，需要引导 | 新手任务、激活激励 |
| 成长用户 | 注册8-30天 | 开始形成使用习惯 | 深度功能引导、打卡活动 |
| 成熟用户 | 注册30天+，周活跃3天+ | 稳定使用，价值用户 | 会员转化、社群运营 |
| 沉默用户 | 7-30天未使用 | 有流失风险 | 召回推送、优惠激励 |
| 流失用户 | 30天+未使用 | 已流失 | 大额优惠、问卷调研 |

## 第三部分：转化漏斗设计

### 一、核心转化漏斗

#### 1.1 新用户激活漏斗

```
【漏斗模型】
曝光 → 点击 → 下载 → 安装 → 启动 → 注册 → 激活 → 首次学习

【关键指标】
曝光量：广告/搜索结果展示次数
点击率（CTR）：点击次数/曝光量 → 目标>3%
下载率：下载次数/点击次数 → 目标>50%
安装率：安装完成/下载开始 → 目标>80%
启动率：首次启动/安装完成 → 目标>90%
注册率：注册完成/首次启动 → 目标>60%
激活率：完成激活/注册完成 → 目标>70%
首学率：首次学习/激活完成 → 目标>80%

【整体转化率】
曝光 → 首次学习：目标>0.5%
```

#### 1.2 付费转化漏斗

```
【漏斗模型】
活跃 → 会员页浏览 → 点击购买 → 支付 → 支付完成

【关键指标】
会员页浏览率：浏览会员页/DAU → 目标>5%
购买点击率：点击购买/浏览会员页 → 目标>20%
支付转化率：支付完成/点击购买 → 目标>30%
整体付费率：付费用户/DAU → 目标>1%

【分群付费率】
新用户（7天内）：目标>0.5%
成长用户（8-30天）：目标>1%
成熟用户（30天+）：目标>3%
高活跃用户（周学5天+）：目标>5%
```

### 二、功能使用漏斗

#### 2.1 学习功能漏斗

```
【场景学习漏斗】
选择场景 → 开始播放 → 播放过半 → 播放完成 → 完成测试 → 添加生词

【指标定义】
开始率：开始播放/选择场景
完播率：播放完成/开始播放
测试参与率：完成测试/播放完成
生词添加率：添加生词/完成测试

【优化方向】
- 开始率低：优化场景选择体验
- 完播率低：优化内容质量/长度
- 测试率低：强化测试引导
- 生词率低：优化生词添加体验
```

#### 2.2 复习功能漏斗

```
【复习漏斗】
提醒触达 → 打开App → 进入生词本 → 开始复习 → 完成复习

【指标定义】
提醒打开率：打开App/收到提醒
生词本进入率：进入生词本/打开App
复习开始率：开始复习/进入生词本
复习完成率：完成复习/开始复习

【优化方向】
- 提醒打开率低：优化推送文案/时间
- 进入率低：强化生词本入口
- 开始率低：简化复习流程
- 完成率低：优化复习体验
```

### 三、社交裂变漏斗

```
【邀请漏斗】
看到邀请入口 → 点击邀请 → 发送邀请 → 好友注册 → 好友学习

【指标定义】
入口曝光率：看到邀请入口/DAU
邀请点击率：点击邀请/看到入口
邀请发送率：发送邀请/点击邀请
邀请成功率：好友注册/发送邀请
邀请激活率：好友学习/好友注册

【关键指标】
人均邀请数：总邀请数/邀请用户数
K因子：邀请成功数/邀请用户数
病毒系数：新用户中来自邀请的比例
```

## 第四部分：A/B测试方案

### 一、测试框架

#### 1.1 测试流程

```
1. 假设提出
   ↓
2. 测试设计（实验组/对照组）
   ↓
3. 样本量计算
   ↓
4. 实验上线
   ↓
5. 数据收集（建议≥7天）
   ↓
6. 结果分析
   ↓
7. 决策与上线
```

#### 1.2 样本量计算

```
公式：n = 16 × σ² / δ²

其中：
- σ：历史数据的标准差
- δ：期望检测的最小差异
- 置信度：95%
- 统计功效：80%

简化为经验值：
- 转化率指标：每组≥1000用户
- 留存率指标：每组≥2000用户
- 收入指标：每组≥3000用户
```

### 二、优先测试项

#### 测试1：新手引导优化

```
【假设】
缩短新手引导流程可以提升激活率

【实验设计】
对照组：现有5步引导流程
实验组：简化3步引导流程

【关键指标】
- 主要指标：激活完成率
- 次要指标：次日留存率、首学完成率

【成功标准】
实验组激活率相对提升>10%
```

#### 测试2：首页布局优化

```
【假设】
将"继续学习"按钮放在首屏可以提升学习启动率

【实验设计】
对照组：现有首页布局
实验组：首屏突出"继续学习"入口

【关键指标】
- 主要指标：每日学习启动率
- 次要指标：人均学习时长

【成功标准】
实验组学习启动率相对提升>15%
```

#### 测试3：付费页面优化

```
【假设】
展示原价划线和限时优惠可以提升付费转化率

【实验设计】
对照组：常规价格展示
实验组：原价划线 + "限时7折"标签

【关键指标】
- 主要指标：付费转化率
- 次要指标：ARPU（每用户平均收入）

【成功标准】
实验组付费转化率相对提升>20%
```

#### 测试4：推送文案优化

```
【假设】
个性化推送文案可以提升打开率

【实验设计】
对照组：统一推送文案
实验组：根据用户学习进度个性化文案

【推送文案示例】
对照组："今天还没学习哦，快来学习吧～"
实验组："你已坚持学习3天，今天继续打卡吗？"

【关键指标】
- 主要指标：推送打开率
- 次要指标：学习完成率

【成功标准】
实验组打开率相对提升>25%
```

#### 测试5：邀请奖励机制

```
【假设】
双向奖励（邀请人和被邀请人都获得奖励）比单向奖励效果更好

【实验设计】
对照组：只有邀请人获得奖励
实验组：邀请人和被邀请人都获得奖励

【关键指标】
- 主要指标：人均邀请数
- 次要指标：邀请成功率

【成功标准】
实验组人均邀请数相对提升>30%
```
