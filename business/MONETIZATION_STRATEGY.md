# EarWords 盈利模式策略

## 一、版本功能划分

### 1.1 免费版 (Free)

| 功能 | 限制说明 |
|------|----------|
| 基础词库 | 仅限小学/初中词汇（约 3,000 词） |
| 每日学习 | 每日最多 20 个单词 |
| 听写模式 | 仅支持基础听写，无智能复习 |
| 发音次数 | 每个单词最多播放 3 次 |
| 离线使用 | 支持，但仅限已下载内容 |
| 广告 | 每次学习结束后显示插屏广告 |
| 数据同步 | 仅支持本机存储，无云端备份 |

**免费版目标**：让用户体验核心功能，建立使用习惯，同时通过限制激发升级欲望。

### 1.2 Pro 版 (Premium)

| 功能 | 说明 |
|------|------|
| 全词库访问 | 覆盖小学至雅思/托福/GRE（50,000+ 词） |
| 无限学习 | 无每日单词数量限制 |
| 智能复习 | 基于艾宾浩斯遗忘曲线的算法 |
| AI 语音识别 | 支持跟读评分与发音纠正 |
| 生词本同步 | 多设备云端同步 |
| 无广告体验 | 完全移除所有广告 |
| 学习报告 | 周/月/年度学习数据分析 |
| 个性化词库 | 支持导入自定义单词列表 |
| 深色模式 | 完整的 UI 主题自定义 |
| 优先客服 | 专属客服通道 |

## 三、订阅 vs 买断模式分析

### 3.1 模式对比

| 维度 | 订阅模式 | 买断模式 |
|------|----------|----------|
| **现金流** | 持续稳定，但依赖续订 | 一次性收入，波动大 |
| **LTV** | 高（优质用户可续订多年） | 固定（一次性） |
| **用户心理** | 门槛高，易取消 | 门槛低，无负担 |
| **开发动力** | 必须持续更新维护 | 易陷入「卖完即弃」 |
| **市场趋势** | 主流（App Store 推荐） | 小众但需求存在 |

### 3.2 EarWords 混合策略

**主推年度订阅 + 保留终身选项**

理由：
1. **教育类 App 适合订阅**：学习是长期行为，订阅符合用户预期
2. **中国用户订阅意愿提升**：Apple One、视频会员培养了习惯
3. **终身版作为兜底**：满足「买断党」需求，避免用户流失
4. **财务预测更可控**：订阅收入可预测性强，便于规划

## 七、收入模型公式

### 7.1 订阅收入预测

```
月度订阅收入 = MAU × 付费率 × ARPU

其中：
- MAU：月活跃用户数
- 付费率：付费用户占比（目标 5-8%）
- ARPU：用户平均收入 = Σ(各档位价格 × 销售比例) / 总付费用户

示例计算（10万 MAU）：
- 付费用户：6,000 人（6% 付费率）
- 销售比例：月度 15% + 年度 60% + 终身 25%
- ARPU = (18×15% + 68×60% + 128×25%) = ¥71.9/人
- 月度收入 ≈ 6,000 × 71.9 / 12 = ¥35,950（首年平滑后）
```

### 7.2 广告收入预测

```
广告收入 = DAU × 人均广告展示 × eCPM / 1000

示例（10万 DAU，免费用户占 94%）：
- 免费用户：94,000 人
- 人均展示：3 次/天
- eCPM：¥40
- 日广告收入 = 94,000 × 3 × 40 / 1000 = ¥11,280
- 月度广告收入 ≈ ¥338,400
```

### 7.3 总收入预估

| 阶段 | MAU | 付费率 | 月度收入（订阅） | 月度收入（广告） | 总计 |
|------|-----|--------|------------------|------------------|------|
| 启动期 | 10,000 | 3% | ¥1,800 | ¥34,000 | ¥36,000 |
| 成长期 | 50,000 | 5% | ¥15,000 | ¥170,000 | ¥185,000 |
| 成熟期 | 200,000 | 8% | ¥96,000 | ¥680,000 | ¥776,000 |

## 八、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 用户订阅意愿低 | 收入不达预期 | 强化终身版选项，降低决策门槛 |
| 竞品价格战 | 用户流失 | 深耕听力细分场景，建立壁垒 |
| 退款率高 | 收入虚高 | 优化首月体验，设置冷静期提醒 |
| 广告体验差 | 用户留存下降 | 控制频次，优先激励视频广告 |
| 苹果政策变化 | 内购受限 | 保持合规，预留网页支付备份 |
