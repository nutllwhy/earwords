## ⚠️ 四、审核风险预案

### 4.1 常见被拒原因及解决方案

#### 4.1.1 功能与描述不符（2.3）

**风险描述：**
应用描述或截图与真实功能不符

**解决方案：**
- 确保描述准确反映当前版本功能
- 截图必须来自真实应用界面
- 不夸大功能效果
- 及时更新描述（功能变更后）

**预防措施：**
- [ ] 描述与功能逐项对照检查
- [ ] 截图使用最新版本录制
- [ ] 避免使用"最"、"第一"等绝对化用语

#### 4.1.2 崩溃或性能问题（2.1）

**风险描述：**
审核过程中遇到崩溃或严重性能问题

**解决方案：**
- 修复所有已知崩溃问题
- 性能优化至可接受水平
- 提供测试账号（如需要登录）

**预防措施：**
- [ ] 多设备真机测试
- [ ] 长时间稳定性测试
- [ ] 弱网环境测试
- [ ] 后台恢复测试

#### 4.1.3 知识产权问题（5.2.3）

**风险描述：**
音频内容、品牌名称等存在侵权风险

**解决方案：**
- 所有音频内容有合法授权
- 不直接使用其他品牌名称
- 版权声明完整

**预防措施：**
- [ ] 音频素材来源清单
- [ ] 授权文件存档
- [ ] 品牌名称商标查询
- [ ] 版权页面完整

#### 4.1.4 隐私政策问题（5.1.1）

**风险描述：**
隐私政策不完整或与数据收集行为不符

**解决方案：**
- 完整隐私政策
- 与数据收集行为一致
- 包含所有必需信息

**预防措施：**
- [ ] 隐私政策模板检查
- [ ] 数据收集行为审计
- [ ] 第三方SDK隐私政策整合

#### 4.1.5 内购绕过（3.1.1）

**风险描述：**
数字内容/服务使用外部支付，绕过 IAP

**解决方案：**
- 所有数字内容使用 Apple IAP
- 无外链支付方式
- 订阅条款清晰

**预防措施：**
- [ ] 支付流程审计
- [ ] 无外部支付链接
- [ ] IAP 商品配置完整

### 4.3 备用方案（被拒后修改计划）

#### 方案 A：快速修复

**适用场景：** 小问题（描述不准确、小bug等）

**时间预期：** 1-3天

**执行步骤：**
1. 定位问题
2. 快速修复
3. 重新提交
4. 审核备注说明修复内容

#### 方案 B：功能调整

**适用场景：** 功能设计与审核指南冲突

**时间预期：** 1-2周

**执行步骤：**
1. 分析审核意见
2. 制定功能调整方案
3. 开发调整
4. 测试验证
5. 重新提交

#### 方案 C：重大修改

**适用场景：** 业务模式或核心功能需调整

**时间预期：** 2-4周

**执行步骤：**
1. 产品策略重新评估
2. 制定新方案
3. 完整开发周期
4. 全面测试
5. 重新提交

#### 方案 D：临时下架

**适用场景：** 无法快速解决的问题

**执行步骤：**
1. 评估影响
2. 制定修复计划
3. 暂时从其他地区上架（如适用）
4. 修复完成后重新申请
