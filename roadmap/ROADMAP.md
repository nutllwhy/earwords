### 1.2 学习提醒优化

#### 当前问题
- 提醒时间固定，不够智能
- 缺乏个性化提醒内容
- 通知点击后跳转不够精准

#### 优化方案
- [ ] **智能提醒时间**: 基于用户历史学习时间推荐最佳提醒时间
- [ ] **情境感知提醒**: 检测到耳机连接时推送"开始磨耳朵"提醒
- [ ] **渐进式提醒**: 
  - 首次提醒: 柔和提示
  - 二次提醒: 进度提醒
  - 最终提醒: Streak 即将中断警告
- [ ] **富媒体通知**: 支持通知内显示单词预览、快捷操作按钮
- [ ] **勿扰模式**: 学习期间自动屏蔽其他通知

#### 提醒策略矩阵
| 场景 | 触发条件 | 提醒内容 | 优先级 |
|------|----------|----------|--------|
| 每日学习 | 用户设定时间 | "今日还有{count}个单词待学习" | P0 |
| 复习提醒 | SM-2计算到期 | "有{count}个单词需要复习" | P0 |

// 积分计算公式
struct RankingScore {
    // 基础分
    let wordsLearned: Int      // 每词 +10分
    let reviewCompleted: Int   // 每次复习 +5分
    let streakDays: Int        // 每天 +20分
    
    // 加成系数
    let accuracyBonus: Double  // 正确率 >90%: 1.2x
    let streakBonus: Double    // 连续7天: 1.5x
    
    var totalScore: Int {
        let base = wordsLearned * 10 + reviewCompleted * 5 + streakDays * 20
        return Int(Double(base) * accuracyBonus * streakBonus)
    }
}
```

└─────────┘                     └─────────┘
```

#### 同步协议
```typescript
// 同步数据模型
interface SyncData {
  userId: string;
  deviceId: string;
  lastSyncTime: Date;
  
  // 增量数据
  delta: {
    words: WordChange[];
    reviews: ReviewLog[];
    settings: UserSettings;
    achievements: AchievementProgress[];
  };
}

// 冲突解决策略
enum ConflictStrategy {
  SERVER_WINS,    // 服务端优先
  CLIENT_WINS,    // 客户端优先
  TIMESTAMP_WINS, // 时间戳优先（默认）
  MERGE          // 合并数据
}
```

### 5.2 架构升级规划

#### 当前架构问题
- ViewModel 过于臃肿
- Repository 层职责不清
- 缺乏中间层（UseCase）
- 依赖注入不完善

#### 目标架构（Clean Architecture）
```
EarWords
├── Presentation (UI Layer)
│   ├── Views (SwiftUI Views)
│   ├── ViewModels
│   └── Components
├── Domain (Business Logic)
│   ├── UseCases
│   ├── Entities
│   └── Repository Interfaces
├── Data (Data Layer)
│   ├── Repositories (Impl)
│   ├── DataSources
│   │   ├── Local (Core Data)
│   │   └── Remote (CloudKit)
│   └── Mappers
└── Core (Frameworks)
    ├── DI Container
    ├── Networking
    └── Utilities
```

#### 迁移计划
| 阶段 | 时间 | 任务 |
|------|------|------|
| 1 | V1.2 | 引入 UseCase 层，提取业务逻辑 |
